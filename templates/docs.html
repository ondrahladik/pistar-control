Autentizace

Chráněné endpointy kontrolují HTTP hlavičku Authorization ve tvaru Bearer tokenu.

Authorization: Bearer muj_token
  • /api/network vyžaduje autorizaci
  • /api/networks vyžaduje autorizaci
  • /api/status vyžaduje autorizaci
  • /api/config vyžaduje autorizaci

Základní odpovědi

  • 200: požadavek proběhl úspěšně
  • 400: chybí nebo je špatný payload
  • 401: neplatný nebo chybějící token
  • 404: neexistující síť nebo endpoint
  • 500: server nedokázal provést změnu
GET /api/networks

Vrátí seznam dostupných sítí podle konfigurace na serveru včetně aliasů pro zobrazení ve webu.

{
  "networks": [
    {
      "id": "host1",
      "label": "Síť 1"
    },
    {
      "id": "host2",
      "label": "Síť 2"
    }
  ]
}
GET /api/status

Vrátí aktuálně přepnuté aktivní připojení včetně aliasu pro UI.

{
  "current_network": "host1",
  "current_network_label": "Síť 1"
}
POST /api/network

Přepne aktivní síť podle jména v JSON payloadu.

{
  "network": "host1"
}
{
  "success": true,
  "current_network": "host1"
}
GET /api/config

Načte aktuální obsah config/app.ini po sekcích a klíčích.

{
  "config": {
    "api": {
      "port": "5000",
      "token": "muj_token"
    }
  }
}
POST /api/config

Uloží nový obsah konfigurace a vrátí finální podobu souboru.

{
  "config": {
    "api": {
      "port": "5000",
      "token": "muj_token"
    }
  }
}
{
  "success": true,
  "config": {
    "api": {
      "port": "5000",
      "token": "muj_token"
    }
  }
}
