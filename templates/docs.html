API

REST API

Autentizace

Chráněné endpointy kontrolují HTTP hlavičku Authorization ve tvaru Bearer tokenu.

Authorization: Bearer <api-token>
  • /api/network vyžaduje autorizaci
  • /api/networks vyžaduje autorizaci
  • /api/status vyžaduje autorizaci
  • /api/config vyžaduje autorizaci
  • /api/hosts/<name> vyžaduje autorizaci

Endpointy

  • GET /api/status: aktuální síť, alias, aktivita a čas
  • GET /api/networks: seznam sítí a aliasů
  • POST /api/network: přepnutí aktivní sítě
  • GET/POST /api/config: čtení a uložení app.ini
  • GET/POST /api/hosts/<name>: práce s host soubory

Typické odpovědi

  • 200: požadavek proběhl úspěšně
  • 400: chybí nebo je neplatný payload
  • 401: neplatný nebo chybějící token
  • 404: neexistující síť nebo endpoint
  • 500: server nedokázal provést změnu

Status payload

{
  "current_network": "host1",
  "current_network_label": "ODS",
  "active_call": {
    "callsign": "OK1ABC",
    "talkgroup": "230"
  },
  "last_update_at": "18:42:11"
}
MQTT

Publikace stavu

JSON payload

Pokud je MQTT zapnuté a je vyplněný broker i publish topic, aplikace publikuje aktuální stav jako JSON.

{
  "network": "BM",
  "callsign": "OK1ABC",
  "talkgroup": "230",
  "time": "18:42:11"
}
  • publikaci zapíná přepínač mqtt.enabled
  • nastavení je v sekci mqtt v config/app.ini
  • pole username a password jsou volitelná
  • pole network obsahuje alias aktivní sítě

Nastavení a příkazy

  • přepínač mqtt.enabled zapíná publikaci i příjem příkazů
  • broker používá server, port a volitelně username, password
  • topic_pub slouží pro stavové JSON zprávy a topic_sub pro příkazy
  • na subscribe topic můžeš poslat alias jako bm nebo interní název host1, bez lomítka
  • odesílá se po přepnutí sítě, změně aktivity a po ukončení hovoru
Telegram

Ovládání bota

Dashboard

Telegram bot udržuje jednu průběžně aktualizovanou zprávu s aktivní sítí, volačkou, talkgroup a časem poslední změny.

  • zapíná se přes telegram.enabled
  • vyžaduje bot_token a chat_id
  • thread_id je volitelné pro topics

Ovládání

  • přepínání funguje přes inline tlačítka i aliasové příkazy
  • aliasy se odvozují z názvů sítí v sekci aliases
  • po změně stavu se dashboard automaticky obnoví
